## Standard Practices
  
Standard on what needs comments and how to format them, ie Class comments, Method comments
  
  ```java
  /*
   * Name: <Class Name>
   *
   * Description: <Why>
   *
   * Related Classes: <Class Names:Relations>
   */
  ```
  
  
  ```java
  /*
   * Name: <Method/Function Name>
   *
   * Description: <Why>
   *
   * Args: <Arg1:Type>, <Arg2:Type>
   *
   * Returns: <Return Type>
   */
  ```
